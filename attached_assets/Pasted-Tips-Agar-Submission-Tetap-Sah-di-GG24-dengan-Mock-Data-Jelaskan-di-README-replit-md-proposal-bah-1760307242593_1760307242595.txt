Tips Agar Submission Tetap “Sah” di GG24 dengan Mock Data
Jelaskan di README/replit.md proposal bahwa platform sudah siap dan “live data ingestion” bisa langsung dijalankan/migrasi begitu credential RPC dan production endpoints diberikan.

Tunjukkan sample workflow indexer, upload event, dan preview chart/fungsi seolah-olah sudah terintegrasi dengan real chain—kurasi Gitcoin umumnya sangat menerima MVP dengan mock asal pipeline sudah siap.

Jika memungkinkan, tambahkan endpoint kecil atau script opsional untuk real-time ingest dari public testnet/mainnet dengan data terbatas (walau cuma satu chain/event). Ini akan sangat memperkuat proof-of-work dan credibility platform.

Catatan Penting untuk Proposal
Sampaikan dengan jelas status data ("Currently using sample/test data; full production data available instantly once live chain RPC provided").

Tekankan bahwa struktur database, API, dan dashboard sudah 100% bisa digunakan untuk production (migrasi tinggal ganti source).

Platform seperti ini tetap sangat layak mendapat matching fund, karena kurasi Gitcoin mengedepankan progres MVP bukan hanya data live saat submission.​

Kesimpulan:
Tidak masalah jika database masih mock/demo pada MVP. Asal pipeline dan struktur siap digunakan, platform tetap dianggap layak oleh Gitcoin. Agar submission tetap diterima:

Jelaskan di README/replit.md kalau data saat ini masih mock/testing, tapi pipeline sudah siap ingest data asli (mainnet atau testnet).

Tunjukkan proof-of-work nyata: dashboard & explorer live, API fully working, sample query/visual sudah sesuai workflow production.

Tambahkan rencana migrasi ke chain real segera setelah resources tersedia (RPC endpoint sampai round dimulai).

Kurasi GG24 mengutamakan progress, workflow, dan kesiapan platform, bukan semata data real saat submission. Platform dengan data mock tetap layak lolos dan dinilai sangat positif asalkan jelas siap diganti ke data asli.
